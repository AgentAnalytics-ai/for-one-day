# Internal documentation

This folder holds **historical and working notes** from development: UX audits, deployment checklists, implementation summaries, and similar material.

- It is **not** required reading to clone, build, or deploy the app.
- For setup and architecture, use the [main README](../../README.md) and [`ENV_SETUP.md`](./ENV_SETUP.md) / [`SETUP.md`](./SETUP.md) as needed.
- Cross-links between files in this folder use **relative paths** within `docs/internal/`.

Keeping these files here keeps the repository root clear for contributors and security reviewers.
